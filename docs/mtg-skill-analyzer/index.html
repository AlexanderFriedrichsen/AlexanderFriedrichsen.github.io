Methodology

Data Sources

Tournament data comes from MTG Meta Analyzer (MIT license), which aggregates match results from melee.gg (tabletop) and MTGO (online) events. Player skill ratings are sourced from MTG ELO Project.

Archetype Classification

Decks are classified into archetypes using signature card matching. Each archetype is defined by a set of required cards and minimum copy counts. Decklists matching all signature cards are assigned to that archetype. Unclassified decks are excluded from the matrix.

Skill Adjustment

The skill-adjusted view accounts for player strength differences using ELO ratings:

  • Each player is matched to their ELO rating from mtgeloproject.net (top ~200 competitive players). Unmatched players default to 1500 ELO.
  • For each match, the expected win probability is calculated from the ELO difference: E = 1 / (1 + 10^((R_opp - R_player) / 400))
  • The residual (actual result minus expected result) is computed. A positive residual means the archetype outperformed what skill alone predicted.
  • The skill-adjusted win rate = 50% + average residual across all matches in that cell.
  • This removes the bias where strong players inflate their deck's apparent win rate.

Limitations

  • ELO data covers only ~200 top professional players. Most tournament players default to 1500 ELO, so skill adjustment is most impactful for formats with many tabletop (melee.gg) events featuring known pros.
  • MTGO tournaments use usernames that cannot be matched to ELO ratings, so those players always default to 1500.
  • Small sample sizes (shown in each cell) mean high variance. Cells with fewer than 5 matches are faded.
  • Mirror matches are excluded from the matrix.

How to Read the Matrix

Each cell shows the win rate of the row archetype against the column archetype. Green = favorable matchup (>50%), Red = unfavorable (<50%). The number below is the match count. The rightmost column shows the archetype's overall non-mirror win rate.
